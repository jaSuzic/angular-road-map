Shared Module

The Shared Module contains reusable components, directives, and pipes that are used across multiple feature modules.

What belongs in Shared?

  • Commonly used components (buttons, cards, modals, etc.)
  • Directives (tooltips, highlight, etc.)
  • Pipes (date formatting, currency, etc.)
  • Commonly used Angular modules (CommonModule, FormsModule, etc.)

Implementation Example

Traditional approach (with NgModule):

<!-- shared/shared.module.ts -->
import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';
import { ButtonComponent } from './components/button/button.component';
import { HighlightDirective } from './directives/highlight.directive';
import { ShortenPipe } from './pipes/shorten.pipe';

@NgModule({
  imports: [CommonModule],
  declarations: [
    ButtonComponent,
    HighlightDirective,
    ShortenPipe
  ],
  exports: [
    CommonModule,
    ButtonComponent,
    HighlightDirective,
    ShortenPipe
  ]
  // No providers here!
})
export class SharedModule { }

Modern approach (Standalone):

<!-- shared/components/button/button.component.ts -->
import { Component, Input } from '@angular/core';
import { CommonModule } from '@angular/common';

@Component({
  selector: 'app-button',
  standalone: true,
  imports: [CommonModule],
  template: `
    <button [class]="'btn ' + type" [disabled]="disabled">
      <ng-content></ng-content>
    </button>
  `,
  styles: [`
    .btn {
      padding: 8px 16px;
      border-radius: 4px;
      cursor: pointer;
    }
    .btn.primary {
      background: #1976d2;
      color: white;
    }
    .btn.secondary {
      background: #f5f5f5;
      color: #333;
    }
  `]
  })
export class ButtonComponent {
  @Input() type: 'primary' | 'secondary' = 'primary';
  @Input() disabled = false;
  }

Best Practices

  • Never provide services in SharedModule (to avoid multiple instances)
  • Keep shared components truly reusable and stateless when possible
  • Use barrel files (index.ts) for cleaner imports
  • Document component inputs/outputs clearly
  • With standalone components, import only what you need

Barrel File Example

// shared/components/index.ts
export * from './button/button.component';
export * from './card/card.component';

// In your component
import { ButtonComponent, CardComponent } from '@app/shared/components';
