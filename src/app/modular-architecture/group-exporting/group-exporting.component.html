Group Exporting in Standalone Components

In the standalone world, you can still logically group components and "export" a single thing (like routes or a Feature API) instead of importing each component individually. Here are the most practical patterns:

1. Export Routes (Recommended)

The entire module is loaded with a single line via lazy-route; the consumer doesn't import components directly, only the route.

Project Structure

2. Feature API Object (Barrel Pattern)

If you want a stronger contract than just an array of routes, export a feature "API" that includes routes, helper providers, and types.

Benefits of This Approach

  • Encapsulation: Components and their dependencies are properly encapsulated
  • Single Responsibility: Each file has a clear, single responsibility
  • Lazy Loading: Enables efficient code splitting and lazy loading
  • Maintainability: Easier to maintain and refactor
  • Testability: Easier to test in isolation
