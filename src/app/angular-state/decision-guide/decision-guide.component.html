Quick decision checklist (pragmatic)

  • Only one component cares? → Component signals.
  • Few components share, basic CRUD, simple async? → Service + signals/RxJS.
  • Feature-scoped but needs effects and structure? → Component Store.
  • App-wide entities, heavy effects, need devtools/traceability? → NgRx.
  • CRUD-heavy domain and you want minimal boilerplate? → Akita.

Progressive enhancement approach:

Start simple and only add structure when pain appears. This decision tree helps you choose the right tool for your current needs.
