Separation of Concerns (Service / Facade / Store / Component)

The component is free from any business logic or state (except for UI state like title, etc.). The Facade provides a "use case" API for the UI, the store maintains state + relationships, and the service communicates with the backend.

Layered Architecture

ApiService

Clean HTTP layer that communicates with the backend.

Store

State management with immutable updates (using signals or BehaviorSubject).

Facade

Orchestrates calls, maps to store, and exposes read-only selectors/signals.

Component

Binds to read-only data and dispatches intents (load/save).
